Создать сетевое устройство (виртуальный интерфейс) после создания пространства имен netns1 можно с помощью команды ip link add, которая создает специальную виртуальную "пару" (veth pair). Она действует как цифровой кабель, соединяющий ваше основное пространство имен с только что созданным netns1 .

Пошаговая инструкция, как это сделать:

Шаг 1: Создайте пару виртуальных интерфейсов (veth pair)
Выполните в терминале команду для создания двух связанных интерфейсов. Например, назовем их veth0 (останется в основной системе) и veth1 (поместим в netns1):

bash
ip link add veth0 type veth peer name veth1
Шаг 2: Переместите один из интерфейсов в пространство имен netns1
Теперь "оторвите" один конец нашего виртуального кабеля (veth1) от основной системы и подключите его к вашему пространству имен:

bash
ip link set veth1 netns netns1
Шаг 3: Настройте IP-адреса и включите интерфейсы
Чтобы интерфейсы начали работать, им нужно назначить IP-адреса из одной сети и включить их.

Для интерфейса в основной системе (veth0):

bash
# Назначаем IP-адрес
ip addr add 10.0.1.1/24 dev veth0
# Включаем интерфейс
ip link set veth0 up
Для интерфейса в пространстве имен netns1 (veth1):

bash
# Выполняем команду ВНУТРИ пространства имен netns1
ip netns exec netns1 ip addr add 10.0.1.2/24 dev veth1
# Включаем интерфейс внутри netns1
ip netns exec netns1 ip link set veth1 up
# Также не забудьте включить петлевой интерфейс (loopback) внутри netns1
ip netns exec netns1 ip link set lo up
Результат и проверка
Теперь ваша основная система и изолированное пространство имен netns1 соединены виртуальной "сетью".

Чтобы проверить соединение, выполните ping из основной системы к netns1:

bash
ping 10.0.1.2
Или наоборот, из netns1 к основной системе:

bash
ip netns exec netns1 ping 10.0.1.1
Если вы все сделали правильно, пинг должен проходить успешно .

Краткая схема команд
Для быстрого повторения, вот минимальный набор команд, который создает сеть между вашей системой и пространством имен netns1:

bash
# 1. Создаем пространство имен
ip netns add netns1

# 2. Создаем пару виртуальных интерфейсов
ip link add veth0 type veth peer name veth1

# 3. Перемещаем один интерфейс в пространство имен
ip link set veth1 netns netns1

# 4. Назначаем IP и поднимаем интерфейсы
ip addr add 10.0.1.1/24 dev veth0
ip link set veth0 up

ip netns exec netns1 ip addr add 10.0.1.2/24 dev veth1
ip netns exec netns1 ip link set veth1 up
ip netns exec netns1 ip link set lo up

# 5. Проверяем связь
ping 10.0.1.2
